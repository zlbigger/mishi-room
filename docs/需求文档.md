# 密室需求文档

## 1. 文档信息

- 产品名称：密室
- 当前版本：v0.3.0
- 产品形态：公网可访问的临时双人互动网站
- 核心定位：一间会消失的双人互动房间
- 目标赛道：生活娱乐

## 2. 产品背景

很多线上交流工具要么太重，要么太公开。情侣、朋友、网友或临时协作对象在短时间互动时，常常只是想一起写点东西、画个草图、贴张图片、传个文件，并不想注册账号、加好友、建群或留下长期记录。

密室希望提供一种更轻、更私密、更有趣的互动方式：发一个链接和口令，对方进入同一个临时房间。双方可以在同一块白板上写、画、贴、传；聊完、画完、传完，就让房间消失。

## 3. 产品目标

### 3.1 核心目标

- 降低临时互动门槛：不注册、不加好友，创建后即可分享。
- 提供轻量共创体验：支持实时画画、文字、贴图、粘贴截图和附件传输。
- 强化私密感和控制权：链接加口令进入，任意一方可销毁，房间到期自动消失。
- 形成有传播力的产品记忆点：让用户感受到“进入一间只属于两个人的密室”。

### 3.2 非目标

- 不做公开社交广场。
- 不做长期文件存储。
- 不做复杂企业级在线白板。
- 当前版本不做账号体系、好友关系、历史房间列表。

## 4. 目标用户与使用场景

### 4.1 目标用户

- 情侣：异地互动、涂鸦、传照片、写小纸条。
- 朋友：快速分享截图、图片、文件和想法。
- 陌生人/网友：不加好友也能临时破冰。
- 临时协作者：快速讨论草图、页面、流程或文件。

### 4.2 使用场景

- 异地情侣晚上一起画今天的心情，并贴几张照片。
- 网友初次聊天时，用一个临时房间破冰，降低尴尬感。
- 朋友临时传一张图片、一个 PDF 或一段说明，不想发到长期聊天记录里。
- 两个人远程讨论一个简单方案，在白板上圈画重点，结束后销毁。

### 4.3 当前痛点

- 加好友、建群、开会议软件流程太重。
- 传统文件传输只能传文件，缺少共同创作和即时标注。
- 普通在线白板偏办公，学习成本高，不适合轻松私密互动。
- 聊天工具里的图片、文件和文字记录长期存在，用户缺少“结束后消失”的控制感。

## 5. 核心价值

### 5.1 用户价值

- 更轻：链接和口令即可进入。
- 更有趣：画、写、贴、传比纯文字聊天更有情绪表达。
- 更安心：房间可销毁、可过期，不做公开列表。
- 更高效：临时协作不需要安装软件或创建复杂空间。

### 5.2 商业价值

- 可扩展为情侣互动、私密社交、轻协作、临时文件共享等多场景产品。
- 后续可探索主题房间、加密房间、会员存储额度、品牌定制房间等商业化方向。

## 6. 功能需求

### 6.1 创建密室

优先级：P0

用户可以在首页创建一个密室，填写房间名称，设置进入口令，选择房间氛围和自动销毁时间。

验收标准：

- 用户输入至少 4 位口令后可以创建房间。
- 创建成功后自动进入房间。
- URL 变为 `/r/:roomId`。
- 房间包含名称、模式、创建时间、过期时间。

### 6.2 口令进入

优先级：P0

收到链接的用户打开房间地址后，需要输入口令和可选昵称才能进入。

验收标准：

- 口令正确时进入房间。
- 口令错误时提示错误。
- 未加入房间时不能订阅实时事件。

### 6.3 实时共享白板

优先级：P0

双方进入同一个房间后，可以实时看到彼此在白板上的操作。

验收标准：

- 支持画笔绘制。
- 支持橡皮擦除。
- 支持文字贴纸。
- 支持图片贴到白板。
- 支持画布拖拽平移和滚轮缩放。
- 一方操作后，另一方可以实时收到事件。

### 6.4 图片粘贴

优先级：P0

用户可以直接把截图、复制来的网页图片或图片链接粘贴到白板。

验收标准：

- 支持剪贴板中的图片文件。
- 支持 HTML 剪贴板中的图片地址。
- 支持常见图片 URL。
- 图片默认贴到用户最近点击或移动的位置。
- 图片超过限制时给出提示。

### 6.5 图片移动与缩放

优先级：P0

用户可以在白板上选中图片，拖动改变位置，也可以拖动图片右下角控制点进行缩放。图片位置和尺寸变化需要同步给房间内其他用户。

验收标准：

- 图片贴到白板后可以被选中。
- 选中图片后可以拖动移动位置。
- 拖动右下角控制点可以按比例缩放图片。
- 移动或缩放结束后，同房间其他用户可以看到更新后的位置和尺寸。

### 6.6 附件传输

优先级：P1

用户可以上传小文件作为临时附件，房间内其他用户可以下载。

验收标准：

- 支持多文件选择。
- 文件显示名称、大小、上传者。
- 文件超过大小限制时跳过并提示。
- 附件在房间销毁后不再可见。

### 6.7 邀请信息

优先级：P0

进入房间后，用户可以复制邀请链接和口令。

验收标准：

- 侧栏显示房间链接和口令。
- 点击邀请或复制按钮后复制完整邀请信息。
- 复制失败时给出可手动复制的反馈。

### 6.8 在线成员

优先级：P1

房间内展示当前在线成员。

验收标准：

- 显示在线人数。
- 显示成员昵称和颜色标识。
- 成员进入或离开时自动更新。

### 6.9 销毁密室

优先级：P0

任意一方可以销毁房间，销毁后双方都会断开连接，房间内容不可再访问。

验收标准：

- 点击销毁按钮后出现二次确认。
- 确认销毁后房间广播销毁事件。
- 所有连接断开并返回首页。
- 房间事件清空。

### 6.10 自动过期

优先级：P0

房间根据创建时选择的有效期自动销毁。

验收标准：

- 页面显示剩余时间。
- 到期后自动触发销毁。
- 过期房间再次访问时提示不存在或已消失。

## 7. 体验与设计要求

- 首屏要让用户立刻理解：这是一个私密、临时、可销毁的互动房间。
- 创建流程控制在 1 个表单内，不增加账号注册。
- 房间页以白板为核心，工具栏保持轻量。
- 视觉风格要有年轻感、趣味性和传播感，避免传统办公白板气质。
- 移动端需要保证工具栏、按钮、文字不挤压、不横向溢出。
- 销毁行为属于高风险操作，必须有明确确认。

## 8. 安全与隐私需求

### 8.1 当前版本

- 房间 ID 使用高强度随机字符串，避免被猜测。
- 房间需要口令才能加入。
- 服务端不提供公开房间列表。
- 房间数据保存在内存中，服务重启后消失。
- 房间销毁后清空事件并断开连接。

### 8.2 后续增强

- 使用口令在浏览器侧派生加密密钥，实现端到端加密。
- 文件进入临时对象存储，设置过期清理。
- 对陌生人模式加入举报、冷静期销毁或安全日志。
- 限制上传文件类型、大小、频率，降低滥用风险。
- 公网部署时启用 HTTPS、速率限制和基础风控。

## 9. 技术需求

### 9.1 当前实现

- 前端：原生 HTML、CSS、JavaScript。
- 后端：原生 Node.js HTTP 服务。
- 实时同步：Server-Sent Events 订阅事件，HTTP POST 提交操作。
- 存储：内存房间 Map。
- 静态资源：Node 服务直接托管 `public/`。

### 9.2 当前限制

- 服务重启后房间全部消失。
- 附件和图片通过 Data URL 保存在事件里，不适合大文件和长期存储。
- 当前没有端到端加密。
- 当前没有账号、权限管理、举报审核。
- 当前没有多人房间和历史版本回放。

## 10. 关键页面

### 10.1 首页

页面目标：让用户理解产品，并快速创建密室。

核心元素：

- 产品名和一句话介绍。
- 产品特点：免注册、口令进入、随时销毁。
- 创建密室表单。
- 进入密室表单。

### 10.2 房间页

页面目标：让双方在同一空间完成写、画、贴、传。

核心元素：

- 顶部房间名称、倒计时、邀请、销毁。
- 左侧或底部工具栏。
- 中央共享白板。
- 右侧邀请信息、在线成员、附件列表和安全规则。

## 11. 成功指标

### 11.1 产品指标

- 创建房间成功率。
- 创建到首次互动的平均时间。
- 邀请复制点击率。
- 双人成功进入率。
- 房间内平均互动事件数。
- 销毁功能使用率。

### 11.2 体验指标

- 用户是否能在 10 秒内理解产品用途。
- 用户是否能在 30 秒内创建并分享房间。
- 新用户是否无需解释即可完成画图、贴图或传附件。

## 12. 版本规划

### v0.3.0 当前版本

- 完成密室品牌和核心互动原型。
- 支持创建、加入、无限白板、白板平移缩放、粘贴图片、图片移动缩放、附件、在线状态、销毁和自动过期。

### v0.4.0 建议下一版

- 增强移动端画板体验。
- 支持清空画布、撤销、下载白板截图。
- 优化图片压缩和上传大小控制。
- 增加房间创建后的分享二维码。

### v0.5.0

- 接入对象存储。
- 增加端到端加密雏形。
- 增加陌生人模式下的安全策略。

### v1.0.0

- 公网 HTTPS 部署。
- 完整过期清理任务。
- 基础风控、限流、监控。
- 稳定的双人房间体验。

## 13. 风险与待确认问题

- 任意一方销毁房间能保护隐私，但也可能被恶意用户用于销毁证据。
- 附件传输需要考虑恶意文件、违规内容和成本控制。
- 如果强调端到端加密，服务端审查能力会下降，需要权衡隐私和安全。
- 浏览器对剪贴板图片支持程度不完全一致，需要兼容主流浏览器。
- 公网部署后需要加入基础防刷和滥用限制。

## 14. 验收清单

- 可以本地运行并打开首页。
- 可以创建密室并进入房间。
- 另一位用户可以通过链接和口令加入。
- 双方可以看到对方绘制的内容。
- 可以在白板上添加文字、图片和粘贴截图。
- 可以上传并下载附件。
- 可以复制邀请信息。
- 可以看到在线人数变化。
- 可以手动销毁房间。
- 房间到期后自动消失。
- 桌面和移动端没有明显布局溢出。
