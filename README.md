# 密室

一间会消失的双人互动房间。

密室是一个轻量、私密、即开即用的公网互动空间。创建者生成链接和口令后，可以发给互联网上的任意一个人。双方进入同一个临时房间后，可以一起写字、画图、贴图、粘贴截图、传附件；任意一方都可以销毁房间。

## 产品一句话

发个链接，进一间密室。写点秘密，画点心情，传点东西，然后让它消失。

## 适合谁用

- 情侣：远程涂鸦、传照片、留小纸条。
- 陌生人：不加好友也能临时破冰互动。
- 朋友：快速共享截图、图片、文件和想法。
- 临时协作：不用注册、不建群，讨论完就销毁。

## 当前版本

`v0.4.0`

- 创建密室，生成分享链接和口令。
- 输入口令进入同一个房间，系统自动分配临时代号。
- 使用开源 Excalidraw 作为白板内核。
- 无限共享白板，支持拖拽平移、滚轮缩放、图形、手绘、文本、便签和图片。
- 支持直接粘贴截图或复制来的图片。
- 图片和图形可在白板上选择、移动、拖拽缩放。
- 支持 Excalidraw 原生快捷键、框选、撤销、缩放和导出体验。
- 支持临时附件传输。
- 在线成员显示，当前用户会高亮自己的随机代号。
- 一键复制邀请信息。
- 任意一方销毁密室。
- 到期自动销毁。

## 设计原则

- 简约：第一屏只做创建和进入。
- 有趣：像一张临时房间票据，而不是复杂办公白板。
- 有用：写、画、贴、传都能马上完成。
- 私密：链接加口令进入，不做公开大厅。
- 可消失：房间内容默认临时存在，手动或到期销毁。

## 本地运行

```bash
npm start
```

默认地址：

```text
http://localhost:4173
```

当前实现无第三方运行依赖，使用原生 Node.js HTTP 服务和浏览器前端。房间数据保存在内存里，服务重启后会消失。

## 自动部署

项目已预留 GitHub Actions 自动部署流程：

- 触发方式：推送到 `master` 分支，或在 GitHub Actions 页面手动触发。
- 目标目录：`/www/wwwroot/mishi.zlbigger.com`
- 服务端口：`4173`
- 服务托管：优先使用 PM2；如果服务器没有 PM2，则自动创建 `systemd` 服务 `mishi-room.service`。
- 反向代理：如果服务器存在 Nginx 配置目录，会生成 `mishi.zlbigger.com` 的 80 端口反向代理配置。

GitHub 仓库需要配置以下 Secrets：

- `MISHI_SERVER_HOST`
- `MISHI_SERVER_USER`
- `MISHI_SERVER_PASSWORD`
- `MISHI_SERVER_PORT`
- `MISHI_DEPLOY_PATH`
- `MISHI_DOMAIN`

## 下一步路线

- 端到端加密：口令只在浏览器侧派生密钥。
- 对象存储：大图、视频和附件进入临时对象存储。
- 房间模式：亲密模式、陌生人模式、协作模式做不同安全策略。
- 举报和冷静期：防止陌生人场景下恶意销毁证据。
- 移动端体验：进一步强化触屏画画和复制粘贴。
- 公网部署：HTTPS、域名、速率限制、文件大小限制和清理任务。
